# אופקי — אבטחה, ביצועים, נגישות

> משימות שחלות על כל השלבים — לבדוק לאחר כל שלב

## אבטחה
- [ ] `NEXT_PUBLIC_` — לוודא שאין מידע רגיש ב-env ציבורי
- [ ] rate limiting על טפסי הגשה — למנוע spam
- [ ] CORS — לוודא שה-API חסום לדומיינים זרים
- [ ] Supabase Storage — bucket פרטי, גישה רק דרך signed URLs
- [ ] SQL injection — Prisma מגן אוטומטית, לא לכתוב raw queries
- [ ] XSS — אין `dangerouslySetInnerHTML` ללא sanitization
- [ ] headers אבטחה — `X-Frame-Options`, `Content-Security-Policy` ב-next.config

## פרטיות
- [ ] קורות חיים — signed URL עם תפוגה (לא URL פתוח קבוע)
- [ ] מועמד לא רואה פרטי מועמדים אחרים — בדיקה כפולה
- [ ] מעסיק לא רואה פרטי מועמדים ישירות — הכל דרך הצוות
- [ ] מחיקת מועמד לפי בקשה — GDPR

## ביצועים
- [ ] תמונות — `next/image` עם `alt` מתאים (לא תמונות אנשים!)
- [ ] Server Components בכל מקום אפשרי — להקטין JS ל-client
- [ ] `loading.tsx` בכל route שעושה שליפת DB
- [ ] `error.tsx` בכל route — הודעת שגיאה ידידותית בעברית
- [ ] Caching — לוח משרות: `revalidate` כל X דקות

## נגישות
- [ ] כל טופס — `<label>` מקושר לכל שדה
- [ ] כפתורים — `aria-label` ברור בעברית
- [ ] ניווט מקלדת — כל האתר נגיש ללא עכבר
- [ ] contrast ratio — לוודא ירוק / אפור על לבן עומד בתקן

## SEO (מינימלי — פרטיות קודמת)
- [ ] `metadata` בכל דף — כותרת + תיאור בעברית
- [ ] `robots.txt` — חסימת `/admin`
- [ ] `sitemap.xml` — דפים ציבוריים בלבד
